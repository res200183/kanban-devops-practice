🚀 DevOps Workflow with GitHub Projects (Kanban)

📌 Overview

This project demonstrates a real-world DevOps workflow using GitHub:

- Issues for task tracking
- Git branches for feature development
- Pull Requests for code review
- GitHub Projects (Kanban) for workflow visualization

---

🧠 Workflow

Backlog → In Progress → Done

Process:

1. Task created as Issue
2. Developer creates branch
3. Commit with "closes #issue_number"
4. Pull Request created
5. Merge → Issue автоматически закрывается
6. Task moves to Done

---

🛠 Tech Stack

- Git
- GitHub
- GitHub Projects (Kanban)
- SSH authentication
